# 机器视觉管理平台概要设计文档

## 1. 系统概述

### 1.1 项目目标

机器视觉管理平台（SLSD Vision Platform）是一个用于管理和展示机器视觉相关数据集与训练模型的Web平台。系统采用B/S架构，实现数据集和模型的统一管理、可视化预览和训练结果分析。

## 3. 功能模块设计

### 3.1 模块划分

| 模块名 | 职责 | 主要功能 |
|--------|------|----------|
| database.py | 数据库管理 | 初始化、CRUD操作、迁移 |
| dataset_manager.py | 数据集管理 | 上传、校验、解析、统计 |
| model_manager.py | 模型管理 | 上传、精度解析、图表生成 |
| annotation_parser.py | 标注解析 | YOLO/VOC/COCO格式解析 |
| storage.py | 存储管理 | 文件读写、目录管理 |
| template_renderer.py | 模板渲染 | HTML模板注入、静态资源服务 |

### 3.2 前端组件

| 组件名 | 职责 |
|--------|------|
| App | 主应用组件，状态管理 |
| 导航栏 | 页面切换、搜索框 |
| 数据集列表 | 数据集卡片展示、筛选 |
| 数据集详情 | 详细信息展示、图表 |
| 模型列表 | 模型卡片展示、筛选 |
| 模型详情 | 精度曲线、PR曲线 |
| 上传弹窗 | 数据集/模型上传 |
| 设置弹窗 | 系统配置管理 |
| 图片预览 | 大图弹窗展示 |

## 4. 数据库设计

### 4.1 表结构

#### 4.1.1 数据集表 (datasets)

```sql
CREATE TABLE datasets (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT UNIQUE NOT NULL,
    algo_type TEXT,
    tech_method TEXT DEFAULT '目标检测算法',
    description TEXT,
    source TEXT,
    split TEXT,
    total INTEGER DEFAULT 0,
    label_count INTEGER DEFAULT 0,
    labels TEXT,
    maintain_date TEXT,
    maintainer TEXT,
    preview_count INTEGER DEFAULT 8,
    annotation_format TEXT DEFAULT 'yolo',
    storage_type TEXT DEFAULT 'folder',
    annotation_type TEXT DEFAULT 'yolo',
    split_ratio TEXT,
    has_test BOOLEAN DEFAULT 0,
    bg_count_train INTEGER DEFAULT 0,
    bg_count_val INTEGER DEFAULT 0,
    bg_count_test INTEGER DEFAULT 0,
    bg_count_total INTEGER DEFAULT 0,
    img_count_train INTEGER DEFAULT 0,
    img_count_val INTEGER DEFAULT 0,
    img_count_test INTEGER DEFAULT 0,
    class_info TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 4.1.2 模型表 (models)

```sql
CREATE TABLE models (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT UNIQUE NOT NULL,
    algo_name TEXT,
    tech_method TEXT DEFAULT '目标检测算法',
    category TEXT,
    accuracy REAL DEFAULT 0,
    description TEXT,
    site TEXT,
    dataset TEXT,
    maintain_date TEXT,
    maintainer TEXT,
    preview_count INTEGER DEFAULT 8,
    model_type TEXT DEFAULT 'yolo',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 4.1.3 设置表 (settings)

```sql
CREATE TABLE settings (
    id INTEGER PRIMARY KEY CHECK (id = 1),
    algo_types TEXT DEFAULT '["路面积水检测","漂浮物检测","墙面裂缝检测","游泳检测","其他"]',
    tech_methods TEXT DEFAULT '["目标检测算法","实例分割算法"]',
    annotation_types TEXT DEFAULT '["YOLO格式","VOC格式","COCO格式"]',
    sites TEXT DEFAULT '["苏北灌溉总渠","南水北调宝应站","慈溪北排","慈溪周巷","瓯江引水","互联网"]',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 4.1.4 原始数据表 (raw_data)

```sql
CREATE TABLE raw_data (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT UNIQUE NOT NULL,
    algo_type TEXT,
    description TEXT,
    dataset TEXT,
    source TEXT,
    is_annotated BOOLEAN DEFAULT 0,
    maintain_date TEXT,
    maintainer TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 5. API设计

### 5.1 数据集API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/datasets | 获取数据集列表 |
| GET | /api/datasets?q=xxx | 搜索数据集 |
| GET | /api/dataset/<name>/images | 获取数据集图片 |
| POST | /api/dataset/upload | 上传数据集 |

| DELETE | /api/dataset/<name> | 删除数据集 |

## 6. 安全设计

### 6.1 输入验证

- 数据集名称：仅允许字母、数字、下划线、连字符
- 文件名：使用werkzeug的安全文件名函数
- 路径：防止目录遍历攻击

### 6.2 数据转义

- 用户输入的特殊字符进行HTML转义
- 防止XSS攻击

### 6.3 文件安全

- 上传文件类型检查
- 文件大小限制
- 危险文件扩展名过滤

## 7. 性能优化

### 7.1 前端优化

- React状态管理减少不必要的重渲染
- 图片懒加载
- 缩略图减少网络传输

### 7.2 后端优化

- 图表缓存机制
- 静态文件Gzip压缩
- 数据库索引优化

## 8. 扩展性设计

### 8.1 模块化

- 各功能模块独立，便于维护和扩展
- 配置与代码分离

### 8.2 可配置项

- 算法类型词条可配置
- 技术方法词条可配置
- 应用现场词条可配置
- 标注格式可配置
