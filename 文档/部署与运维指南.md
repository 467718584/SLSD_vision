## 3. 部署模式

### 3.1 开发/测试环境

```bash
# 直接运行
python server.py

# 访问 http://localhost:8501
```

### 3.2 生产环境 (Docker)

#### 方式一：使用 Docker Image (推荐)

```bash
# 1. 拉取镜像
docker pull sldsvision/platform:latest

# 2. 创建数据目录
mkdir -p ~/sldsvision/data

# 3. 运行容器
docker run -d \
  --name sldsvision-platform \
  -p 8501:8501 \
  -v ~/sldsvision/data:/app/data \
  sldsvision/platform:latest
```

#### 方式二：使用 Docker Compose

```bash
# 1. 克隆项目
git clone <repo-url>
cd SLSD_vision

# 2. 启动服务
docker-compose up -d

# 3. 访问 http://<server-ip>:8501
```

## 4. 版本更新流程

### 4.1 更新原则

1. **数据隔离**: 所有数据存储在 `data/` 目录，与代码分离
2. **数据库迁移**: 使用数据库迁移机制，不影响已有数据
3. **向后兼容**: 新版本兼容旧版本的数据结构

### 4.2 更新步骤

#### 步骤1：备份数据（重要！）

```bash
# 备份整个数据目录
tar -czvf sldsvision-backup-$(date +%Y%m%d).tar.gz data/

# 或仅备份数据库
cp data/vision_platform.db data/vision_platform.db.backup
```

#### 步骤2：更新代码

```bash
# 方式一：使用 Docker
docker-compose down
git pull
docker-compose build --no-cache
docker-compose up -d

# 方式二：直接更新（如果已部署）
git pull
docker build -t sldsvision/platform .
docker stop sldsvision-platform
docker rm sldsvision-platform
docker run -d \
  --name sldsvision-platform \
  -p 8501:8501 \
  -v $(pwd)/data:/app/data \
  sldsvision/platform:latest
```

#### 步骤3：验证

- 访问 http://<server-ip>:8501
- 检查数据是否完整
- 测试核心功能

### 4.3 回滚流程

```bash
# 如果需要回滚
docker-compose down
# 恢复备份
tar -xzvf sldsvision-backup-20260317.tar.gz
# 使用旧版本镜像或代码
git checkout <previous-tag>
docker-compose up -d
```

## 6. 数据库迁移机制

### 6.1 迁移流程

```python
# modules/database.py

def migrate_database(conn, cursor):
    """数据库迁移"""
    # 1. 检查现有表结构
    cursor.execute("PRAGMA table_info(datasets)")
    columns = {row[1] for row in cursor.fetchall()}

    # 2. 添加新字段（如果不存在）
    new_columns = {
        'source': 'ALTER TABLE datasets ADD COLUMN source TEXT',
        'site': 'ALTER TABLE models ADD COLUMN site TEXT'
    }

    for col, sql in new_columns.items():
        if col not in columns:
            try:
                cursor.execute(sql)
            except:
                pass  # 忽略可能的错误
```

### 6.2 迁移特性

- **自动执行**: 每次启动时检查并迁移
- **向后兼容**: 新字段有默认值
- **幂等性**: 多次执行不会出错

## 7. 运维监控

### 7.1 日志查看

```bash
# Docker 容器日志
docker logs sldsvision-platform

# 实时日志
docker logs -f sldsvision-platform

# docker-compose 日志
docker-compose logs -f
```

### 7.2 健康检查

```bash

curl http://localhost:8501/api/datasets
```

## 8. 故障排查

### 8.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 页面空白 | Docker构建失败 | 检查日志 |
| 数据丢失 | 未使用volume挂载 | 重新挂载data目录 |
| 无法访问 | 端口被占用 | 检查8501端口 |
| 中文乱码 | 字体问题 | 使用正确的基础镜像 |

### 8.2 数据恢复

```bash
# 1. 停止容器
docker-compose down

# 2. 恢复数据
rm -rf data/*
tar -xzvf sldsvision-backup-20260317.tar.gz

# 3. 启动容器
docker-compose up -d
```
